Presentation Tips — Jump Search

- Start with the list sorted and label indexes clearly.
- Explain the goal: jump ahead in blocks instead of checking every value.
- Use a step size like `sqrt(n)` and show how it reduces the number of comparisons.
- Demonstrate one or two jumps, then switch to a linear scan inside the block.
- Emphasize that the array must be sorted for Jump Search to work.
- Compare with Linear Search: Jump Search skips blocks, then scans only one block.
- Mention the runtime idea: O(√n) because of block jumps plus one linear scan.
- Keep examples small, such as 7–10 elements, to show each jump clearly.
- End with a practice question: find a target in a sorted list using block jumps and a final scan.
